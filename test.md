	Which language is the best programming language is one of the most controversial problems for all of the programmers. 
  In addition, a new language, Rust, becomes a competitive one in this war as soon as it was published.
  Rust is a systems programming language focused on three goals: safety, speed, and concurrency. (Rust Documentation) 
  By Offering low-level memory manipulation and a reliable safety, the Rust language grabs an enormous number of “crazy” fans from different developer groups.
  The user of Rust includes from system programmer to the web developer (Cburgdorf).
  They all have a great passion to explore this amazing language. Before Rust, most people believe that the safety is a tradeoff to control the low-level memory for the high-level programming language.
  For example, C and C++ provides a strong control to the memory, but it is very essay to crash in the program which is written by C and C++. 
  However, Rust solves this problem perfectly by the ownership and lifetime model. 
  While retaining the safety, rust is also a good high-level object oriented programming language. 
  The Rust do not give up any property to achieve its goals. 
  It still supports objects, generics, functional programming and etc. 
  Based on these properties, I argue that Rust has a great potential to be a major programming language in the near future.
